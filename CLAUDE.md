# 國中會考線上系統

**⚠️ 開始任何工作前,先讀 [HANDOFF.md](HANDOFF.md)** — 裡面有完整的開發進度、架構、腳本、測試帳號與重要教訓。

讀完 HANDOFF.md 後,需要更細的規劃再看 `docs/01~07`。

每完成一個里程碑,請更新 HANDOFF.md 底部的「進度日誌」。

## 開工 / 收工 關鍵字

使用者說「**開工**」:
1. 讀 HANDOFF.md(尤其開頭「新對話第一件事」區塊,確認目前實際運行狀態,不要照單全收——涉及外部狀態的
   描述,如伺服器是否在跑、部署是否成功、IP 是否還一樣,要動手驗證而非只信任文件記載,文件會過期)。
2. 之後所有開發依 HANDOFF.md / docs/01~07 記載的規範與既有架構原則進行(不要無視既有慣例另起爐灶)。

使用者說「**收工**」:
1. 把這次對話做的事總結寫進 HANDOFF.md 底部「進度日誌」(新的一行,加在最上面,格式比照既有日誌:做了
   什麼、為什麼、遇到並解決的問題、下一步)。
2. 如果運行狀態有變化(部署網址、伺服器狀態、密碼、IP、分支狀態等),同步更新開頭「新對話第一件事」區塊,
   讓下一個對話不用重新調查一次就能拿到正確現況。
3. 只更新檔案,**不要自動 commit/push**(除非使用者當下另外明確要求)——commit 與 push 是各自獨立的動作,
   要分開明確同意。
