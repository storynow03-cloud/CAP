# 08 — 商業化、版權與 AI 出題評估

> 結論先講:**只要開始靠這個系統賺錢(廣告/贊助),就不能再用康軒題庫**。
> 但「用 AI 生成你自己的原創題庫」完全可行,而且是這類產品擴大的標準解法。

## 1. 版權現況盤點

| 內容 | 版權歸屬 | 自用(家人) | 商業化(廣告/贊助) |
|------|---------|:---:|:---:|
| 康軒題庫(精選題/習作/素養題…) | 康軒出版社 | ⚠️ 灰色(技術上仍是重製,但個人風險低) | ❌ **侵權** |
| 國中會考真題 | 教育部/心測中心,**公開** | ✅ | ✅(教育用途,仍建議標註來源) |
| AI 生成的原創題 | **你自己** | ✅ | ✅ |

**關鍵法律點:**
- 「贊助」即使不是賣商品,只要系統「整體是營利性質」,使用他人版權內容就很難主張合理使用。
- 私下給自己孩子用,跟「公開上網+收費」是兩個世界 — 後者等於公開散布。
- 結論:**要商業化,就得把康軒題目換掉**,只留會考真題 + 自製/AI 題。

## 2. AI 出題 — 可行嗎?可行,而且很適合

你的題庫已經有最寶貴的資產:**課綱代碼(學習內容 N-7-3 等)、單元、難度標記**。這正是餵給 AI 生成「對齊課綱」題目的最佳輸入。

### 出題管線(用 Claude API)
```
指定:科目 + 單元 + 課綱代碼 + 難度(1~5)+ 題型
  → Claude 生成:題幹 + 4 選項 + 正解 + 詳解(繁中、國中程度、會考風格)
  → 自動驗證:格式(4選項/單一正解)、用第二次 AI 核對答案是否正確
  → 標 needs_review,人工抽查通過才上線
  → 存進同一個 questions 表(無縫接軌現有系統)
```

### 品質與風險(誠實說)
- **數學最容易出錯**:AI 可能算錯或答案標錯 → 必須「生成 + 第二次 AI 驗算 + 人工抽查」三道關。
- **避免含圖題**:純文字 AI 不擅長幾何圖,先生成「文字型」數學/自然題,圖表題另議。
- **避免抄到版權題**:提示明確要求「原創、不要複製既有考題」。
- **成本**:用 API 一題成本約幾分錢,生成幾千題只要幾百元台幣,極划算。
- **課綱對齊**:用你現有的課綱代碼當約束,品質會比憑空生成好很多。

### 結論
AI 出題是**最務實的商業化基礎** — 你擁有完整著作權、可無限擴充、成本極低。建議商業化版本以「**會考真題(公開)+ AI 生成原創題**」為題庫主體。

## 3. 商業化要實作 / 改變的東西

| 項目 | 說明 | 優先 |
|------|------|:--:|
| **題庫換成自有** | 移除康軒題,改用會考真題 + AI 生成題 | ★★★ 必須 |
| **隱私權政策 + 服務條款** | 你收集 email、學習紀錄 → 受**個資法**規範,必須有 | ★★★ 必須 |
| **未成年人保護** | 使用者是國中生 → 需家長同意機制、兒少資料保護 | ★★★ 必須 |
| **金流(贊助)** | 接綠界/街口/Stripe,放「贊助我們」按鈕 | ★★ |
| **廣告** | Google AdSense/AdMob;但**兒少教育 App 放廣告要很小心**(版位、內容過濾、隱私) | ★ 不建議優先 |
| **Supabase 升級** | 免費方案有用量上限,使用者變多要升付費 | ★★ 視規模 |
| **內容審核** | 若開放使用者互動(大會考、留言),需審核機制 | ★ |

### 我的建議路線
1. **現在**:自用、私人,康軒題庫先測試系統沒問題。
2. **要公開前**:先做「AI 原創題庫」替換康軒題(我可以幫你建出題管線)。
3. **變現方式**:**先做「贊助」不做廣告** — 對教育/兒少 App 比較乾淨、不傷體驗、法律單純。
4. **法律文件**:上線前一定要有隱私權政策、服務條款、家長同意 — 這部分建議諮詢律師或用合規範本。

> ⚠️ 本文件是技術與商業面的整理,**不是法律意見**。實際商業化前請諮詢專業律師,特別是版權與個資、兒少保護部分。
