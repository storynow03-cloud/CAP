# 04 — 開發路線圖

原則:每一階段結束都有「可以動手測試的東西」,你驗收後才進下一階段。
全程程式碼只存本地 `D:\Claude\國中會考`,直到你同意才上 GitHub。

## 階段 0:準備(現在)
- [x] 架構文件(本批文件)
- [ ] 你放入題庫檔案到 `data/raw/`(任何格式皆可)
- [ ] 我轉換題庫成標準 JSON,你抽查確認題目、答案、單元、難度標得對不對
- [ ] 建立 Supabase 免費專案(需要你註冊帳號,我會一步步帶)

## 階段 1:最小可用版本(MVP)
**目標:能登入、能做題、有紀錄**
- 建立 Next.js 專案 + Supabase 串接(登入/註冊)
- 題庫匯入資料庫
- 自由練習模式(選科目/單元/難度 → 作答 → 對答案看詳解)
- 作答紀錄寫入資料庫
- ✅ 驗收:你登入後做完一回題目,重新整理紀錄還在

## 階段 2:分階挑戰(核心)
- 等級制度 + 出題引擎(排除已精熟、優先弱點)
- 回合內自適應難度
- 升降階判定與動畫
- ✅ 驗收:連續答對會明顯感覺題目變難、不再出現做過的簡單題

## 階段 3:錯題本與學習歷程
- 錯題本 + 間隔複習排程
- 儀表板:會考倒數、今日目標、連續天數
- 學習歷程:雷達圖、進步曲線、單元弱點分析
- ✅ 驗收:故意答錯幾題,隔天出現在待複習;報表數字正確

## 階段 4:模擬會考
- 整卷計時模式(依會考題數配置)
- 模擬等級換算(A++~C)
- ✅ 驗收:完整模擬一科,計時與成績換算正確

## 階段 5:上線
- `git init` → 推上 GitHub(私人 repo,需你同意)
- Vercel 匯入部署 + 環境變數設定
- 手機版面檢查(學生大多用手機)
- ✅ 驗收:用手機開網址,登入做題正常
