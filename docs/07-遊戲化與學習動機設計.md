# 07 — 遊戲化與學習動機設計

> 目標:把「每天考試看分數」變成「忍不住想打開、想再做一題」的體驗。
> 參考:Duolingo(每日任務/連勝/寶石商城)、蛋仔派對(角色養成/好友對戰/賽季)。

## 設計原則

1. **即時回饋**:每答一題都有看得到的成長(XP、金幣、進度條動畫)。
2. **每天有理由回來**:每日任務 + 連勝 + 限時活動。
3. **自我表達**:角色/主題/寵物個人化,孩子想「秀」給同儕看。
4. **社交競爭與合作**:跟兄弟姊妹/同學 PK、排行、組隊。
5. **不傷害學習本質**:獎勵綁定「真正的學習行為」(答對難題、複習錯題、攻克弱點),不是刷簡單題。

---

## 核心循環(Core Loop)

```
打開 App → 看每日任務 → 做題(分階挑戰/錯題複習)
   → 得 XP + 金幣 + 連勝 +1 → 升級/解任務/開寶箱
   → 用金幣買裝扮/道具、養寵物 → 想跟好友比 → 隔天再來
```

---

## 功能模組(依「動機效果 / 開發成本」分三階段)

### 🟢 第一階段:個人成長循環(成本低、效果高)

| 功能 | 說明 | 資料 |
|------|------|------|
| **XP 與個人等級** | 每題給 XP(答對難題加成、複習錯題加成),累積升「玩家等級」(與學科等級分開)。等級條動畫。 | `profiles.xp`, `level` |
| **金幣** | 完成任務/答對得金幣,商城消費用。 | `profiles.coins` |
| **每日任務** | 每天 3 個小任務,如「完成 15 題」「複習 5 題錯題」「挑戰數學 1 回合」。完成給 XP+金幣。 | `daily_quests` |
| **連勝強化** | 現有連續天數 + 「補簽卡」(漏一天可補)。連勝里程碑給獎勵。 | 現有 `daily_stats` + 道具 |
| **成就徽章** | 「連續 7 天」「單科精熟」「答對 100 題」「攻克 50 錯題」等,解鎖徽章牆。 | `achievements` |

### 🟡 第二階段:個人化與養成(中成本)

| 功能 | 說明 |
|------|------|
| **商城** | 金幣購買:角色裝扮、主題色、頭像框、道具(提示卡、雙倍 XP 卡、補簽卡)。 |
| **學習夥伴(寵物)** | 蛋仔風格的小夥伴,做題「餵養」會成長/進化,陪考倒數時給打氣。可換造型。 |
| **個人主頁** | 展示等級、徽章、寵物、最佳紀錄 — 可分享給好友看。 |

### 🔴 第三階段:社交與競技(高成本、高黏著)

| 功能 | 說明 |
|------|------|
| **好友系統** | 加兄弟姊妹/同學為好友,看彼此進度、互相打氣。 |
| **1v1 對戰(PK)** | 同一份題目比「又快又準」,即時或非同步。贏得對戰積分。 |
| **週排行榜** | 好友間 / 班級內的「本週 XP 榜」,每週重置。(現有大會考排行可延伸) |
| **賽季 + 轉蛋** | 每月一賽季,累積賽季點數換限定造型;寶箱/轉蛋開裝扮(只給造型,不影響學習公平)。 |
| **王關 Boss 戰** | 每週一隻「魔王」= 一組主題難題,打倒給大獎(個人或組隊)。 |

## 與現有系統的銜接

- **已有的基礎**:分階挑戰、錯題本、連續天數、大會考排行、雷達圖 — 這些都是遊戲化的「行為來源」,只要在上面疊「獎勵層」。
- **獎勵綁定學習品質**:XP 公式建議 `基礎分 × 難度係數 × (錯題複習×1.5 / 弱點單元×1.3)`,讓孩子自然往「該練的」走,而不是刷簡單題。

## 防沉迷與健康

- 每日任務有「上限」,完成就好,不鼓勵無限刷。
- 寵物/獎勵走「穩定陪伴」而非「賭博式刺激」(轉蛋只給造型、機率透明)。
- 家長端可看「學習時間 vs 遊戲化互動」比例。

---

## 建議第一步(MVP 遊戲化)

只做第一階段的四件事,大約能讓黏著度明顯提升,且都能複用現有作答資料:
1. XP + 個人等級條(首頁顯眼處)
2. 每日 3 任務
3. 金幣 + 一個簡單商城(先賣主題色/頭像框)
4. 成就徽章牆

> 等孩子實際用一陣子、確認願意每天開,再投入第二、三階段的養成與社交。
