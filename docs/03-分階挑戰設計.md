# 03 — 分階挑戰設計(核心機制)

設計目標:**不讓學生一直重複做簡單題**,持續挑戰能力邊緣,並把錯的東西練到會。

## 1. 等級制度

每科 × 每單元各自有等級(不是整科一個等級,這樣弱點單元才會被照顧到):

| 等級 | 名稱 | 出題難度組成 | 對應會考 |
|:--:|------|------------|---------|
| Lv1 | 入門 | 難度1 為主 | 待加強 → 基礎 |
| Lv2 | 基礎 | 難度1-2 | 基礎 |
| Lv3 | 進階 | 難度2-3 | 基礎 → 精熟 |
| Lv4 | 精熟 | 難度3-4 | 精熟(B++~A) |
| Lv5 | 挑戰 | 難度4-5 | A~A++ |

**升階條件**:該等級最近 10 題答對率 ≥ 80% → 升階(顯示升階動畫激勵)。
**降階保護**:最近 10 題答對率 < 40% → 退一階鞏固,避免挫折感。

## 2. 出題引擎規則(每回合 10 題)

依優先序組卷:

1. **錯題複習(最多 3 題)**:錯題本中到期該複習的題目
2. **弱點單元(約 4 題)**:該科中等級最低 / 答對率最差的單元,出當前等級難度
3. **當前推進(約 3 題)**:目前正在挑戰的單元,難度 = 當前等級,**排除 30 天內答對過 2 次以上的題目**(這就是「不重複做簡單題」的關鍵)

題目不足時自動放寬條件遞補,並提示「此單元題庫不足」。

## 3. 回合內自適應

- 連對 3 題 → 下一題難度 +1(提前試探更高等級)
- 連錯 2 題 → 下一題難度 -1(穩住信心)
- 試探題答對會加速升階判定

## 4. 錯題本與間隔複習(Spaced Repetition)

```
答錯 → 進錯題本,1 天後到期
複習答對 → 下次間隔 ×2(1天 → 3天 → 7天 → 14天)
連續答對 3 次 → 移出錯題本(標記已克服)
複習又答錯 → 間隔重設回 1 天
```

儀表板每天顯示「今日待複習 N 題」,複習優先於新題。

## 5. 精熟度計算

每個「科目 × 單元」維護一個精熟度分數(0~100):

```
精熟度 = 加權答對率(最近的作答權重高) × 難度係數
```

- 答對難題加分多、答錯簡單題扣分多
- 雷達圖、弱點排行、出題引擎都讀這個分數

## 6. 激勵機制

- 會考倒數天數(首頁大字)
- 每日目標(預設 20 題)+ 連續達標天數 🔥
- 升階徽章、單元精熟徽章
- 每週學習報告(本週題數、正確率變化、進步最多的單元)
