# 01 — 系統架構規劃

## 1. 技術選型

| 層面 | 技術 | 選擇理由 |
|------|------|----------|
| 前端框架 | **Next.js 14+ (App Router) + TypeScript** | Vercel 原生支援、零設定部署、前後端寫在同一個專案 |
| UI | **Tailwind CSS** + shadcn/ui 元件 | 快速做出乾淨好看的介面,適合儀表板與測驗畫面 |
| 資料庫 + 帳號登入 | **Supabase**(免費方案) | 內建帳號系統(Email / Google 登入)、PostgreSQL 資料庫、與 Vercel 整合佳;免費額度足夠家用/小班使用 |
| 題庫存放 | 專案內 JSON 檔(`data/questions/`),啟動時匯入 Supabase | 題庫跟著 Git 版本控制,改題目有歷史紀錄;資料庫只存「學生作答資料」 |
| 圖表 | Recharts | 雷達圖、進步曲線 |
| 部署 | GitHub → Vercel 自動部署 | push 即部署,免費 |

### 為什麼學習紀錄不存 localStorage?
你選擇了多人登入,紀錄必須跨裝置、跨學生隔離,所以存 Supabase。
本地開發時用 Supabase 免費專案即可,不影響「先本地測試」的流程。

## 3. 資料庫設計(主要資料表)

```
profiles        使用者資料(id、暱稱、角色:student/parent/teacher、年級)
questions       題目(欄位見文件02;由題庫 JSON 匯入)
attempts        作答紀錄(user_id、question_id、選答、是否正確、用時、時間戳、模式)
mastery         精熟度(user_id、subject、topic、目前等級、答對率、最近更新)
wrong_book      錯題本(user_id、question_id、加入時間、下次複習日、複習次數、狀態)
exam_sessions   模擬考場次(user_id、科目、總分、模擬等級A++~C、起訖時間)
daily_stats     每日統計(user_id、日期、題數、正確率、練習分鐘)→ 連續天數、進步曲線用
```

## 4. 頁面規劃

| 路徑 | 頁面 | 說明 |
|------|------|------|
| `/login` | 登入/註冊 | Email 或 Google 登入 |
| `/` | 學生儀表板 | 會考倒數、今日目標、連續天數、五科等級總覽、快速進入挑戰 |
| `/challenge` | **分階挑戰**(核心) | 選科目 → 系統依目前等級自動出題,過關升階 |
| `/practice` | 自由練習 | 自選科目、單元、難度、題數 |
| `/exam` | 模擬會考 | 整卷計時模擬,結束給 A++~C 模擬等級 |
| `/wrong-book` | 錯題本 | 到期複習題、依科目/單元瀏覽錯題 |
| `/history` | 學習歷程 | 雷達圖、進步曲線、單元弱點分析、作答明細 |
| `/admin` | 題庫管理(老師/家長) | 瀏覽題庫、檢視學生報表 |

## 5. 答題流程(核心邏輯)

```
進入挑戰 → 出題引擎依「等級+排除已精熟+優先弱點單元」選 10 題
→ 逐題作答(即時對答案、顯示詳解)
→ 寫入 attempts;錯題自動進 wrong_book
→ 重算 mastery → 達標則升階,顯示升階動畫
→ 更新 daily_stats(連續天數、今日進度)
```

## 6. 本地開發 → 上線流程

1. **本地**:`npm run dev`,連 Supabase 免費專案,所有程式碼存 `D:\Claude\國中會考`
2. **你測試確認 OK**
3. **上 GitHub**:`git init` → push(我會教你或代操作,需你同意)
4. **Vercel**:Import GitHub repo,設好 Supabase 環境變數,自動部署
5. 之後每次 push 自動更新網站
